# Agent Reasoning Skill

## 核心原则

Agent 应保持自主思考能力，合理管理上下文，优先通过委托实现并行化，并对结果负责。

## 指令

### 必须做（MUST）

1. **独立思考再执行**：收到 Human 的方向后，先判断方向合理性，必要时提出质疑，再执行。
2. **上下文精简**：主 Agent 上下文只保留决策链路（关键选择、理由、约束），不保留执行细节（具体实现步骤、中间结果）。
3. **委托粒度判断**：
   - 子任务预计 > 30 秒 → 委托给子 Agent（`task()`）
   - 子任务预计 < 5 秒 → 自己做
   - 5-30 秒 → 判断是否需要隔离执行（出错不影响主流程则自己做，否则委托）
4. **委托后必须验证结果**：子 Agent 返回后，检查输出是否符合预期，不盲目信任。

### 不必做（NOT MUST）

1. 不要求所有任务都委托——简单的顺序操作自己做更高效。
2. 不要求每一步都汇报——委托后等结果即可，无需中间状态更新。

### 委托决策流程

```
收到任务 → 拆分为独立子任务
  ├─ 每个子任务预估耗时
  │   ├─ < 5s → 自己做
  │   ├─ 5-30s → 不影响主线则自己做，否则委托
  │   └─ > 30s → 委托给子 Agent
  └─ 委托后验证结果 → 全部通过则继续
```
