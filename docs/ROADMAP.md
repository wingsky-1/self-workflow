## V1.0~V1.18 — 核心功能雏形（✅ 已完成）

**一句话**：10 个系统模块，19 个迭代版本，全部由框架自身的工作流开发完成（自举）。

### 10 个已交付系统模块

| # | 系统 | 作用 | 一句话 |
|---|------|------|--------|
| 1 | Installer（安装器） | 部署 | MANIFEST 驱动的声明式部署，模板源→运行时 |
| 2 | /feat 工作流 | 执行 | 5 阶段 + 4 Gate 结构化开发流程 |
| 3 | Review Agent（审查） | 质量 | 只读对抗性审查，Grill+COT 策略逐项挑战 |
| 4 | Task 管理 | 追踪 | task.yaml 生命周期 + Git tag checkpoint + Compound 归档 |
| 5 | Experience（经验管理） | 记忆 | 5 分类经验库 + 5 级生命周期 + exp-governance 治理 |
| 6 | Spec（规范） | 约束 | 8 项 default spec，约束主 Agent 及全部子 Agent |
| 7 | Session Plugin（会话插件） | 注入 | 双钩子自动注入（docs 索引 + specs 摘要）+ 4 个内置工具 |
| 8 | Todo 管理 | 规划 | 三层待办（todo.md→task.yaml→todowrite）+ P0/P1/P2 优先级 |
| 9 | /catchup | 恢复 | 扫描进行中任务和未解决错误，生成恢复摘要 |
| 10 | Config（配置） | 模板 | 11 个阶段模板 + feat-workflow 指引 + task 模板 |

> 详细系统文档见 `docs/README.md` 及 `docs/systems/` 下各子系统文档。

| 里程碑 | 版本 | 关键交付 |

### V1 阶段的两个系统性教训

从 V1.5 自举评审（27/38=71%）暴露，持续影响后续版本：

1. **检查清单 ≠ 强制执行**：Agent 把"清单有这一项"等同于"完成了"——V1.5.1 的 Gate 强制步骤、V1.21 的 Gate 纪律强化，本质上在反复修补同一个问题。
2. **文档型修复不可靠**：写了文档 ≠ Agent 会遵守。V1.3（决策是额外工作）和 V1.6（目录职责混淆）都是 V1 用文档修了、V1.5 Agent 照样重现。→ 催生了 V1.8 的 Spec 系统（注入 system prompt 而非依赖 Agent 主动读）。

## V2.x — 多 Agent 并行（📋 规划中）

**目标**：从"一个 Agent 串行干活"升级为"多个 Agent 并行协同"。

### 与 V1 的核心变化

| 维度 | V1（当前） | V2（目标） |
|------|-----------|-----------|
| 执行模式 | 主 Agent 串行执行全部 5 阶段 | 各阶段委托专用子 Agent，可并行 |
| 会话模型 | 单任务串行 | git worktree 多任务并行 |
| 工作流类型 | 1 种（feat） | 4 种（feat / debug / doc / review） |
| 经验晋升 | Phase 5 手动判断 | Compound 自动晋升 draft→verified |
| 规范来源 | 手动编写 | 经验→spec 自动晋升管道 |

### 核心方向

- **子 Agent 架构**：每阶段专用 Agent/Skill，主 Agent 负责编排和 Gate 审查
- **多会话并行**：git worktree 支持同一项目多任务同时开发
- **经验复利闭环**：Compound 自动晋升 + freshness 监控（verified 文档 30 天自动提醒重审）
- **经验→spec 晋升管道**：经多次验证的参考模式自动发布为 spec/template
- **提示词注入场景梳理**：主 Agent/子 Agent 注入内容精准匹配目标

### 前置条件

- [ ] V1.19~V1.24 收尾完成（Gate 纪律、工具链规范到位）
- [ ] 子 Agent 委托的可靠性经过验证（当前阶段执行仍由主 Agent 完成）

## V3.x — 用户体验（💡 远期）

**目标**：从"能用"到"好用"——减少人工介入，提升交互自然度。

### 核心方向

| 方向 | 说明 |
|------|------|
| **无人值守模式** | /feat 自主完成全流程，用户事后评审 |
| **自然语言触发** | 普通对话中识别工作流意图，无需显式敲 `/feat` |
| **老项目蒸馏** | 已有项目文档自动转 .self-workflow/docs 格式 |
| **智能化决策** | 评审问题置信度 >95% 时自动抉择，减少 question 打断 |
| **用户体验功能** | 周报/日报 command、doctor 诊断命令、安装 patch 预览确认 |
| **平台化** | 自定义工作流 DSL、多 Agent 适配（Cursor/Copilot 等）、团队共享 |

> 远期愿景中已识别的具体项见 `.self-workflow/todo.md` 的 `## Vx：远期愿景` 段。

## 版本对比

| 维度 | V1.x | V2.x | V3.x |
|------|------|------|------|
| **核心价值** | 会话不丢、错误不重犯 | 多 Agent 并行、经验复利 | 从「能用」到「好用」 |
| **Agent 模型** | 主 Agent 串行 | 主 Agent 编排 + 子 Agent 并行 | 自适应分配 |
| **工作流数量** | 1（feat） | 4（feat/debug/doc/review） | 不限（DSL 自定义） |
| **经验系统** | 手动沉淀 + exp-governance 治理 | 自动晋升 + freshness 监控 | 平台化共享 |
| **规范来源** | 手动编写 8 项 spec | 经验→spec 晋升管道 | 社区/团队共享 |
| **交互方式** | Slash Command 显式触发 | Slash Command + 部分自动 | 自然语言 + 无人值守 |
| **安装器** | MANIFEST 声明式部署 | 无损升级 + 选择性安装 | 模板市场 + 多工具适配 |
