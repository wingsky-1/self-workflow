## 路线图总览

```
V1 (已完成) ──→ V1.5 (当前) ──→ V2 ──→ V3+
记住上一次      质量加固         经验复利  平台化
                2026 Q3        2026 Q4   2027+
```

| 版本 | 核心主题 | 一句话描述 |
|------|---------|-----------|
| **V1** | 记住上一次 | 安装器 + feat 工作流 + Review Agent + Catchup → **已交付** |

> **版本调整说明**：原 ROADMAP 的 V2（经验复利）拆分为 V1.5（质量加固）+ V2（经验复利）。四专家联合评审确认：在 8 个已知流程缺陷上构建经验系统会导致"垃圾进垃圾出"。质量必须先于功能。

## V1.5 — "质量加固"（当前，2026 Q3）

**目标**：修复 V1 暴露的 8 个流程缺陷，让工作流达到"生产可用"标准。

**状态：🔜 待实现**

### 背景：8 个流程缺陷

| # | 缺陷 | 影响 | 来源 |
|---|------|------|------|
| 1 | 线性阶段不支持回溯 | 复杂任务卡死 | V1 回顾 |
| 2 | 缺少方向判断 Gate | 架构偏离 | V1 回顾 |
| 3 | 决策记录是额外工作 | ADR 遗漏 | V1 回顾 |
| 4 | Gate 重量模糊 | 行为不可预期 | V1 回顾 |
| 5 | Quick Mode 无法升级 | 规模误判 | V1 回顾 |
| 6 | 缺少引用工件 | 上下文不完整 | V1 回顾 |
| 7 | 模板分批提取 | 效率低 | V1 回顾 |
| 8 | task.yaml 非强制 | 元数据缺失 | V1 回顾 |

### 核心设计

三层质量防线（详见 `docs/V1.5-质量保障需求设计.md`）：
- **流程防错**：方向审查 Gate + Checkpoint 回溯 + 决策嵌入 + Gate 重量量化
- **Agent 约束**：Skills + 检查清单 + Review Agent 行为审查
- **产物验证**：结构化验证 + 交叉引用检查 + 模板强制

### 里程碑分解

| 里程碑 | 功能 | 对应缺陷 | 预计周期 |
|--------|------|---------|---------|
| **M0** | 框架融入：`interaction-protocol` Skill、`agent-reasoning` Skill、`/adr` Command | #3, #8 | 1 周 |
| **M1** | 方向 Gate + Checkpoint 回溯 + 决策捕捉嵌入 | #1, #2 | 2 周 |
| **M2** | 检查清单 + Review Agent 行为审查 + 质疑流程 | #4, #5 | 2 周 |
| **M3** | 结构化验证 + 交叉引用 + 模板强制 | #6, #7 | 1 周 |
| **总计** | V1.5 完整交付 | 8/8 缺陷 | 6 周 |

### 验收标准

- [ ] `/adr` 命令可在 30 秒内创建标准 ADR 文件，自动更新 task.yaml
- [ ] 方向审查 Gate 能识别与现有 ADR 不一致的方案
- [ ] Checkpoint 回溯允许从阶段 N 返回阶段 M（跳过已通过 Gate）
- [ ] 每个阶段末尾自动执行决策捕捉
- [ ] 阶段采用检查清单驱动，消除 Agent 行为歧义
- [ ] Review Agent 审查报告中包含行为审查（behavior）维度
- [ ] 产物模板必填字段缺失时 Gate 不通过
- [ ] **自举可用**：用修复后的工作流在自身上完成一个功能开发，全流程跑通

## V2 — "经验复利"（2026 Q4）

**目标**：每次工作流让下一次同类任务更简单。经验可沉淀、可检索、可信赖。

**前置条件**：V1.5 验收通过（工作流达到生产可用标准）

### 设计要点

- **Compound 经验沉淀**：工作流完成时自动提取经验草稿（draft）
- **经验质量分级**：draft / verified / promoted / elevated
- **全工作流覆盖**：debug / doc / review 工作流
- **Spec 晋升系统**：项目自有 spec → 验证有效 → 晋升到 installer/specs/ 通用化
- **Adapter V2**：编译目标 Skill / Command / Agent 定义

### 验收标准

- [ ] 工作流完成时自动提取经验草稿
- [ ] Review Agent 可将 draft 审查晋升为 verified
- [ ] 四种工作流均可通过 Slash Command 触发
- [ ] V1/V1.5 安装可无损升级到 V2

## V3+ — "平台化"（2027+）

**目标**：让 Self-Workflow 适应任意项目类型和团队规模。

### 方向

| 方向 | 说明 | 优先级 |
|------|------|--------|
| **自定义工作流 DSL** | 用户可定义自己的阶段、Gate 规则和产物模板 | P0 |
| **多 Agent 适配** | 支持 Cursor、Copilot、Claude Code 等 | P1 |
| **团队协作** | 共享 Spec、经验库、工作流模板 | P1 |
| **工作流模板市场** | 社区贡献的预置工作流 | P2 |

| 维度 | V1 | V1.5 | V2 | V3+ |
|------|----|------|-----|-----|
