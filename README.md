# Self-Workflow

> 它不说话。却什么都记得。
>
> *It never speaks. Yet remembers everything.*

---

## 背景

AI 能写代码。但每一次新会话，它都像第一天——不记得约定、不记得教训、不记得三个月前的选择。

不是它笨，是结构缺失。

Self-Workflow 填这个缺口。用自己的框架开发自己——`/feat` 命令、审查 Agent、经验库，每一个都诞生于同一条工作流。

每一次翻车，沉淀为下一次的护栏。

存在就是证明。

它只认三条。

---

## 信念

**结构，不是控制。**

**记忆，不是笔记。**

**省心，不是省脑子。**

---

## 边界

它不替你想、不替你选、不替你写。

它只认三条。

---

## 开始使用

```bash
npx self-workflow init
```

然后：

```
/feat 实现用户登录模块
```

走一遍：分析 → 设计 → 实现 → 验证 → 总结。Gate 在哪里拦了你，经验在哪里被悄悄加载了。

**接下来，改成你的。** 直接编辑 `.self-workflow/`——改写 docs 里的经验，补充你学到的；调整 specs 里的规范，让它贴近你的习惯。创造你自己的 agent、command、skill。

这个框架不教你怎么工作。它只做一件事。

**你的经验不会白费。**

---

## 路线图

**今天**

一条命令启动开发工作流。五扇门——每一扇门前，Agent 停下来审视自己的方向。

一份会生长的经验库。五个分类，五级生命。每一次翻车都变成文档，下一个会话自动翻阅。

八项规范。约束主 Agent，也约束它派出去的所有子 Agent。

十个系统模块。安装器、工作流、审查、任务、经验、规范、插件、待办、恢复、配置——每一个都由框架自身的工作流开发完成。

**明天**

让 Agent 学会分工。多个子 Agent 并行干活，你做你的设计、我写我的代码，主 Agent 坐在中间编排。

让经验学会自己长大。从草稿到验证，从模式到规范——不用人催。

**更远**

把老项目的知识蒸馏出来。让 Agent 不用你敲命令，自己就知道该开始了。让它在说话之前，先想一想——这个问题该问你，还是该自己决定。
