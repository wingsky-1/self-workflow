# Self-Workflow

> 它不说话。却什么都记得。
>
> *It never speaks. Yet remembers everything.*

---

## 背景

AI 能写代码。但每一次新会话，它都像第一天——不记得约定、不记得教训、不记得三个月前的选择。

不是它笨，是结构缺失。

Self-Workflow 填这个缺口。用自己的框架开发自己——`/feat` 命令、审查 Agent、经验库，每一个都诞生于同一条工作流。

每一次翻车，沉淀为下一次的护栏。

存在就是证明。

它只认三条。

---

## 信念

**结构，不是控制。**

**记忆，不是笔记。**

**省心，不是省脑子。**

---

## 边界

它不替你想、不替你选、不替你写。

它只认三条。

---

## 开始使用

```bash
npx self-workflow init
```

然后：

```
/feat 实现用户登录模块
```

走一遍：分析 → 设计 → 实现 → 验证 → 总结。Gate 在哪里拦了你，经验在哪里被悄悄加载了。

**接下来，改成你的。** 直接编辑 `.self-workflow/`——改写 docs 里的经验，补充你学到的；调整 specs 里的规范，让它贴近你的习惯。创造你自己的 agent、command、skill。

这个框架不教你怎么工作。它只做一件事。

**你的经验不会白费。**

---

## 路线图

**今天**

一条命令启动开发工作流：分析、设计、实现、验证、总结。五扇门——每一扇门前，Agent 停下来审视自己的方向。

一份会生长的经验库。每一次翻车都变成文档，下一个会话自动翻阅。

一套规范。同时约束主 Agent 和它派出去的所有子 Agent。

**明天**

让 Agent 自己判断什么时候该问你、什么时候该自己决定。让经验自己检查自己有没有过时。让多个子 Agent 并行干活。

**更远**

把老项目的知识蒸馏出来。给 Agent 一个健康检查。让它不用你敲命令，自己就知道该开始了。
