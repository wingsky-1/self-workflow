# 安装器三层架构与自举认知陷阱

## 根因：三层架构认知缺失

本项目是自举项目——Agent 同时扮演开发者和用户。关键是区分三层：

| 层 | 目录 | 修改方式 |
|----|------|---------|
| 开发层 | `docs/`、`packages/` | 直接编辑 |
| 平台集成层 | `.opencode/` | **仅安装器** |
| 运行时层—configs/specs | `.self-workflow/configs/`、`.self-workflow/specs/` | **仅安装器**（模板源在 `packages/installer/templates/`） |
| 运行时层—数据 | `.self-workflow/tasks/`、`.self-workflow/docs/` | 直接编辑 |

Agent 容易混淆 `.self-workflow/` 下不同子目录的归属——看到 `.self-workflow/` 就认为是"用户数据"可随意改，但 `configs/` 和 `specs/` 实际是安装器的"编译产物"。

## 模式一：执行时错误——直接编辑运行时文件

**案例**：Agent 直接编辑 `.self-workflow/configs/guides/feat-workflow.md`，后续 `init --force` 时被模板源覆盖，修改全部丢失。

**正确做法**：

```
需改 .self-workflow/configs/ 或 .self-workflow/specs/ 下的文件
  ↓
先改 packages/installer/templates/ 下的对应模板源
  ↓
运行 node packages/installer/index.js init --target . --force
  ↓
验证运行时文件内容正确
```

**判断标准**：修改前问自己——这个文件在 `packages/installer/templates/` 下有对应的模板源吗？
- **有** → 改模板源 → 运行安装器
- **无** → 直接改

## 模式二：设计时盲区——可定制性声明未验证

**案例**：ADR-002 声称 feat-workflow.md "用户可定制工作流"。对抗性审查指出该文件在 MANIFEST 中——直接编辑会被覆盖。ADR 需补上"定制方式为修改模板源→运行安装器同步"的限定条件才通过 Gate 2。

**根因**：设计阶段将"逻辑上可定制"等同于"实际上可安全修改"，忽略了安装器运行时约束。

**三步检查**（宣称某文件"用户可编辑/可定制"前）：

1. 该文件在 MANIFEST 中吗？
   - 是 → 定制方式 = 改模板源 + `init --force`
   - 否 → 可直接编辑
2. 安装器是 `init`（覆盖）还是 `init --force`（强制覆盖）？
3. 存在 `--keep-user-config` 保护模式吗？→ 本项目无 → 必须明确标注

**修正示例**：

> 修改前：用户可修改阶段定义、Gate 条件、Checkpoint 规则
>
> 修改后：用户可通过修改安装器模板源 `packages/installer/templates/configs/guides/feat-workflow.md` 定制工作流，运行 `init --force` 同步。直接编辑部署副本会被安装器覆盖。

## 统一判断清单

**执行前**（改任何文件前）：
- [ ] 该文件在 `packages/installer/templates/` 下有对应模板源吗？
  - 有 → 改模板源 → `init --force`
  - 无 → 直接改

**设计时**（宣称某文件"用户可修改"前）：
- [ ] 该文件在 MANIFEST 中吗？
  - 是 → 必须标注"修改模板源→安装器同步"
  - 否 → 可直接编辑

## MANIFEST 覆盖速查

| 运行时路径 | 模板源路径 |
|-----------|-----------|
| `.opencode/agents/*` | `packages/installer/templates/agents/` |
| `.opencode/commands/*` | `packages/installer/templates/commands/` |
| `.opencode/skills/*` | `packages/installer/templates/skills/` |
| `.self-workflow/configs/*` | `packages/installer/templates/configs/` |
| `.self-workflow/specs/*` | `packages/installer/templates/specs/` |
