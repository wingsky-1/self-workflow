# Gate强制步骤实施经验

> 当 Agent 反复跳过框架规范步骤时，仅凭措辞增强无法根治——需三管齐下。

## 核心发现

当 Agent 反复跳过框架规范步骤时，仅凭措辞增强无法根治——需三管齐下：

### 三管齐下模式

| 维度 | 策略 | 示例 |
|------|------|------|
| **措辞** | 提示性问题 → 存在性断言 | "是否需要 ADR？" → "检查 adrs/ 目录——有决策则必有文件" |
| **位置** | 共享引用 → 执行点自包含 | 量化公式从单一定义块复制到 4 个 Gate 入口 |
| **兜底** | 无 → Compound 事后修复 | Compound 扫描缺失 tag → git log --grep 反查补建 |

### 适用场景

此模式适用于所有 "Agent 已被告知某规则但执行时跳过" 的框架缺陷修复。

### 局限

- 措辞改为断言后，Agent 仍可能声称 "本阶段无架构决策" 绕过（需结合 Review Agent 审查）
- 自包含复制增加维护负担（5 处同步），但在公式稳定时维护成本低
- 兜底逻辑仅能修复、不能阻止（tag 缺失后补建 vs 事前确保创建）
