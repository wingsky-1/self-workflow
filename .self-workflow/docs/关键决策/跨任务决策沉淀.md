# 跨任务决策沉淀与 ADR 治理

## 两级存储

```
tasks/<id>/adrs/  ──Phase 5 晋升──→  docs/关键决策/
（决策产生地）                          （持续服务项目）
```

## 晋升流程（Phase 5 自动触发）

1. Agent 扫描本任务所有 ADR
2. 按标准评估跨任务价值
3. `question` 工具向 Human 提议
4. Human 确认 → 写入 `docs/关键决策/`

## 晋升标准

| 标准 | 门槛 |
|------|------|
| 跨任务性 | ≥2 个独立任务引用或依赖 |
| 框架级 | 涉及 `configs/` 或 `installer/` 架构 |
| 持久性 | 非 hotfix / workaround 临时方案 |
| 可引用性 | 其他任务需要此上下文 |

## ADR 生命周期

```
创建 → 引用（同任务互引 + Gate 审查） → 沉淀（晋升） → 废弃（superseded）/ 更新（superseded-by）
```
