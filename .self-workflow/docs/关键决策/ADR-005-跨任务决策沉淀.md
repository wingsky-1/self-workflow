# ADR-005：跨任务决策沉淀 —— docs/关键决策/ 目录 + 人工决策晋升

## 背景

ADR 仅存在于 `tasks/<id>/adrs/` 中（任务级），任务完成后埋没，其他任务无法检索复用。29 个已有 ADR 面临同样问题。

## 备选方案

- **A**：不做跨任务沉淀 ❌ — 越积越多的死数据
- **B**：docs/关键决策/ + Agent 提议 + Human 决策 ✅ 选中
- **C**：自动晋升（按引用次数）❌ — 实现复杂

## 决策

**选择方案 B**：`docs/关键决策/` 目录 + Phase 5 人工决策晋升。

### 流程

```
1. 任务执行中 → ADR 创建在 tasks/<id>/adrs/（不变）
2. Phase 5 总结时 → Agent 扫描 ADR，评估跨任务价值
3. Agent 使用 question 工具向 Human 提议晋升
4. Human 确认 → ADR 复制到 docs/关键决策/
5. 后续：废弃/更新的 ADR 在 frontmatter 标注 status: superseded
```

### 晋升评估标准（Agent 参考）

- **跨任务性**：被 ≥2 个独立任务引用或依赖
- **框架级**：修改了 `.self-workflow/configs/` 或 `packages/installer/`
- **持久性**：非 hotfix/workaround 类临时决策
- **可引用性**：其他任务需要此决策上下文

## 理由

1. **解决真问题**：ADR 可检索性从零到有
2. **不过度自动化**：人工决策避免误判
3. **与经验目录统一**：关键决策/ 作为 docs/ 第四分类，Plugin 自动索引
