# 产物权威来源唯一原则 — ADR引用而非内联

## 核心发现

当同一决策内容出现在多个产物文件中时（如设计文档和独立 ADR 文件），应遵循 **"权威来源唯一 + 引用"** 原则：完整内容只在 adrs/ 目录的独立文件中，其他产物通过摘要表 + 链接引用。

## 反模式（应避免）

```
02-design.md                  adrs/ADR-001.md
┌─────────────────┐          ┌─────────────────┐
│ ## ADR-001       │          │ # ADR-001        │
│ 背景：...        │  重复！   │ 背景：...        │
│ 备选方案：...    │ ←──────→ │ 备选方案：...    │
│ 选择：...        │          │ 选择：...        │
└─────────────────┘          └─────────────────┘
```

问题：修改决策内容需要同步两处，迟早会不一致。

## 正确模式

```
02-design.md                  adrs/ADR-001.md
┌──────────────────────┐     ┌──────────────────────┐
│ ## 架构决策记录       │     │ # ADR-001 (完整内容)  │
│                      │     │ 背景、备选方案、      │
│ | ADR | 决策 | 选择|  │     │ 选择、理由、后果      │
│ | 001 | xxx  | x   |──┼────→│                      │
│ | 002 | yyy  | y   |  │     └──────────────────────┘
│                      │
│ > 完整内容见 adrs/    │
└──────────────────────┘
```

设计文档仅保留决策摘要表（ID、决策、选择、要点 + 文件链接），完整内容在 adrs/ 独立文件中。

## 判断标准

产物文件中是否出现了 ADR 的完整内容（背景、备选方案表、选择理由）？
- ✅ 出现在 adrs/ 目录 → 正确（权威来源）
- ✅ 以摘要表 + 链接形式出现在设计文档 → 正确（引用）
- ❌ 完整内容同时出现在两个文件 → 重复，需精简

## 关联

- F2 规范：阶段结束时 adrs/ 下必须有 ADR 文件——这使 adrs/ 成为 ADR 的唯一权威来源
- `task.yaml` 的 `cross-refs`：同样遵循"引用外部文档而非内联"原则
- 本原则与 "Agent 不会跳转读取引用"（ADR-001 的背景）存在 tension——但适用于不同层：
  - **执行层**（Agent 需要立即执行的操作）：自包含（如 Gate 量化公式复制到执行点）
  - **产物层**（维护权威来源的文档）：引用（如 ADR 内容只在 adrs/ 中）
