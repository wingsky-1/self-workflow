# READMe.md 作为框架配置的权威源模式

## 摘要

用 README.md 同时服务人类读者和机器解析：人类从顶部读取"如何使用"，Plugin 从底部"分类定义"段解析结构化数据。新增分类只需在 README 中添加一行 `### 新分类名/`，Plugin 自动识别——零代码修改、零配置同步。

## 场景

需要一种"分类→文档"的映射机制，既能让人类用户直观浏览，又能让 Plugin 程序化读取。传统做法是维护独立的配置文件（如 `index.yaml`），但配置文件与人类文档不同步风险高。

## 方案

```
README.md
├── # 标题 + ## 如何使用          ← 人类优先
├── ## 文档格式                    ← 人类 + 工具
├── ## Tag 约定                    ← 规范
└── ## 分类定义                    ← Plugin 解析区
    ### 实施经验/
    描述文本                        ← ← ← 正则匹配这里
    ### 参考模式/
    描述文本
```

Plugin 解析规则：
- 正则 `/###\s+(.+?)\s*\/\s*\n+([^\n#]+)/g`
- 提取分类名（`实施经验`）→ 目录名
- 提取描述文本 → 注入到 Agent 系统提示中的分类说明

## 优势

1. **单一权威源**：人和 Plugin 读取同一份文件，不存在配置同步问题
2. **零维护扩展**：新增分类只需加一行标题 + 描述，Plugin 无需改代码
3. **渐进式**：顶部的"如何使用"等人类文档不影响底部的机器解析

## 适用场景

- 需要人类可读 + 机器可解析的配置
- 分类/目录结构需要动态扩展的场景
- 希望减少配置文件数量的项目
