# 验收验证的双重检查：存在 + 不存在

## 问题

验收标准的验证通常只做"正向检查"——确认期望的内容**存在**：

```
✅ "feat.md 步骤 3 引用了 feat-task.yaml"
```

但这不能保证变更的完整性。对于**替换/删除/提取**类变更，正向检查通过但目标未达成的典型场景：

| 变更类型 | 正向检查（通过） | 实际问题 | 应做的反向检查 |
|----------|-----------------|---------|---------------|
| 模板提取 | 新文件存在 | 旧位置还保留着完整内联副本 | 旧位置**不存在**被提取的内容 |
| 目录清理 | README 已创建 | 死代码目录还在 | 旧路径 Test-Path 返回 False |
| 功能移除 | 新行为生效 | 旧代码/配置残留 | 旧模式搜索返回 0 结果 |

## 模式

### 验收标准写法

每条"替换/删除"类验收标准应写为一对检查：

```markdown
- [ ] **A2a**：Given `/feat` 执行时，When 步骤 3 执行，Then 从 `.self-workflow/configs/tasks/feat-task.yaml` 加载模板。
- [ ] **A2b**：Given 模板源文件，When 搜索内联 task.yaml 模板，Then `packages/installer/templates/commands/feat.md` 中不存在 `name: <slug>` 等模板字段。
```

A2a 是正向（引用存在），A2b 是反向（内联已移除）。

### 验证命令模式

```powershell
# 正向：检查期望存在
Select-String -Path feat.md -Pattern "feat-task.yaml"   # 返回 match → ✅

# 反向：检查不应存在
Select-String -Path feat.md -Pattern "name: <slug>"      # 返回空 → ✅ 内联已移除
```

### Gate 验证时的自检问题

在标记验收标准为 PASS 前，问自己：

> 这条变更是否涉及"删除"或"替换"旧内容？
> 如果是 → 我是否同时检查了"新内容存在"**和**"旧内容不存在"？

## 适用场景

- 文件/代码提取（extract to new file）
- 目录/文件清理（delete dead code）
- 配置迁移（old config → new config）
- 模板替换（inline → external template）
- 任何涉及"移除"操作的变更
