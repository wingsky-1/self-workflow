## 经验沉淀

### doc 级经验

本次任务产生了以下可跨任务复用的经验：

1. **"实现方案"文档分类模式**：当项目核心特性达到一定复杂度时，创建独立的实现方案文档（而非仅靠实施经验）能显著降低 Agent 重复读源码的成本。此模式已被框架化（installer 模板），用户 init 即可获得。

2. **spec 引导优于工作流嵌入**：将 Agent 行为引导规则放在独立 spec 中（Plugin 自动注入）优于硬编码到工作流文档中——更易复用、更利于子 Agent 架构转型。

### task 级经验

- 面对框架级特性，Phase 1 应更早引入 Human 确认深层期望（本次需求分析修订 3 轮）
