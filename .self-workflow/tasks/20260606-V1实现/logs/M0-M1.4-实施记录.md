## M0：OpenCode 能力审计

### 实施过程

1. **调研 OpenCode 平台能力**
   - 查阅 oh-my-openagent 源码中的 QA 参考资料（events-hooks.md, server-api.md, cli-commands.md）
   - 查阅 customize-opencode skill 了解 agent/command/skill 定义格式
   - 实验验证：通过 `task()` 创建只读子 Agent 测试 sub-agent 机制

2. **逐个验证架构假设**
   - Hook 机制：发现 OpenCode 的 hook 是 Plugin API 驱动的，不是文件目录驱动的。不存在 `on:session-start` 文件级 hook
   - Sub-agent：通过 `task()` API 验证了子 Agent 可读文件、输出结构化 YAML、保持只读
   - Slash Command：确认支持 `.opencode/command/<name>.md` 格式
   - Skill 系统：确认需 `name` frontmatter 才能加载
   - 文件权限：验证通过

### 关键发现

| 假设 | 实际 | 影响 |
|------|------|------|
| 文件级 lifecycle hook | Plugin API hook，非文件驱动 | Session Catchup 需改为 slash command |
| `.opencode/agents/*.yaml` | `.opencode/agents/*.md`（Markdown + frontmatter） | Review Agent 格式调整 |
| `on:session-start` 自动触发 | 不支持 | V1 用 `/catchup` 手动触发 |

### 产出

- `docs/audits/opencode-capabilities.md`（后迁至 docs/）
