## 实施内容

### 1. `/catchup` 命令定义

创建 `.opencode/commands/catchup.md`，定义 slash command `/catchup`。

**命令流程**：
1. 扫描 `.self-workflow/tasks/` 下所有 `task.yaml`
2. 检测 `in_progress` / `stuck` / `cancelled` 状态的任务
3. 扫描未解决的错误（`errors.yaml` 中 `resolved: false`）
4. 按模板格式生成 Catchup 摘要
5. 询问用户是否恢复某个任务

### 2. 验证

通过 `task()` 调用验证了 catchup 逻辑：

| 验证项 | 结果 |
|--------|------|

| 里程碑统计（5/7 完成） | ✅ |
| 错误检查（无未解决错误） | ✅ |
| 生成结构化摘要 | ✅ |
