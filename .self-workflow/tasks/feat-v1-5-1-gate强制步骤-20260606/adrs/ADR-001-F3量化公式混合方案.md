## 背景

F3 要求每个 Gate 入口强制计算 scope+risk+user-signal 三维分值并显式输出。量化公式本身是通用定义（3 维度 × 3 分值 → 3 权重阈值），可集中定义一次后由 4 个 Gate 引用，也可在每个 Gate 入口独立复制。

## 备选方案

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A: 复制 | 量化公式完整复制到 4 个 Gate 入口 | Agent 无需跳转，执行率最高 | 修改公式需同步 4 处 |
| B: 引用 | 公式定义在共享位置，其他 Gate 引用 | 单一来源，维护成本低 | Agent 实践表明跳转引用可能被跳过 |
| C: 混合 | 附录保留规范定义（canonical），4 个 Gate 入口各含简化计算指令 + 引用附录 | 兼顾两者 | 文档复杂度增加 |

## 选择：方案 C（混合）

## 理由

1. Agent 执行实践反复验证：跨章节引用会被 Agent 跳过（如 Gate 3/4 未量化——本次修复的根因之一）
2. 公式本身短小（~10 行），复制维护成本低
3. 附录保留规范定义确保未来修改时有单一校验源
4. 每个 Gate 入口的计算指令是自包含的——Agent 读一个 section 就能完成量化

## 后果

- ✅ 每个 Gate 入口可独立执行量化计算
- ⚠️ 修改公式需同步附录 + 4 个 Gate 入口（5 处），但在公式稳定（3 维度 × 3 分值 × 3 阈值）的情况下几乎不需要修改
- ✅ 附录提供交叉校验：如果某个 Gate 的计算与规范不符，Review Agent 可检测
