# ADR-003：F2 ADR 检查 — "显式声明"防绕过

## 背景

F2 要求阶段结束时如有决策则 adrs/ 下必须有 ADR 文件。Review Agent (Gate 1) 审查时指出：Agent 可通过声称"本阶段无架构决策"绕过 ADR 创建，而不留下任何痕迹。需要增加防绕过机制。

## 备选方案

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A: 仅改措辞 | 检查清单从"是否需要 ADR？"改为"检查 adrs/ 目录——有决策则必须有文件" | 最小改动 | 仍存在"声称无决策"的绕过空间 |
| B: 显式声明 | A + 要求 Agent 在每个阶段完成时显式声明"本阶段无架构决策"或引用对应 ADR 文件路径 | 留下审计痕迹，可事后检查 | 增加 Agent 执行负担 |
| C: 自动化检查 | 在 Compound 阶段扫描 artifact 文件内容，自动检测是否有决策内容被提及 | 无 Agent 绕过空间 | 实现复杂，依赖 NLP——V2 考虑 |

## 选择：方案 B（显式声明）

## 理由

1. 方案 B 在不增加复杂度的前提下，留下了可审计的痕迹——Review Agent 或人类可检查"无决策"声明是否合理
2. 方案 C（自动化检查）需要 NLP 能力，属于 V2 范围
3. 声明格式简单（一句话），Agent 执行负担可忽略

## 显式声明格式

```
决策捕捉状态：[ ] 本阶段无架构决策  /  [x] ADR 已创建，见 adrs/ADR-XXX-xxx.md
```

## 后果

- ✅ 每个阶段的决策状态可追溯
- ✅ Review Agent 可审查"无决策"声明是否合理（方向性选择、多方案对比等决策场景不应被标记为"无决策"）
- ⚠️ Agent 仍可能错误标记——但此时会在 Review Agent 的 behavior 维度中暴露
- ⚠️ **Review Agent 扩展计划**：当前 Review Agent 仅有笼统的"决策捕捉检查"，未细化到检查显式声明的合理性。此机制依赖 Review Agent 的同步更新才能发挥完整效力——计划在 V1.5.2 或 V1.5.3 中扩展 Review Agent 审查规则，增加"检查 '无架构决策' 声明是否合理"的专项审查。
