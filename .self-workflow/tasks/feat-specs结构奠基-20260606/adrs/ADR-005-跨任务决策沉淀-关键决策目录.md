# ADR-005：跨任务决策沉淀 —— docs/关键决策/ 目录

## 背景

当前 ADR 仅存在于 `tasks/<id>/adrs/` 中（任务级），任务完成后 ADR 埋没在任务目录中，其他任务无法检索和复用。例如 ADR-001~008（V1 实现任务）、ADR-001~004（feat-实现feat命令）中有大量跨任务有价值的决策，但除非 Agent 主动扫描所有历史任务，否则无法发现。

同时，ADR lifecycle 只有"创建"，缺少"沉淀→晋升→废弃/更新"的管理。

Human 反馈：晋升机制不明确的情况下先保持主动询问用户，由用户决策。

## 备选方案

### 方案 A：不做跨任务沉淀

- ADR 仅保留在任务目录中
- Agent 需要时手动扫描历史任务查找

❌ 已有 29 个 ADR 埋没，越积越多的死数据

### 方案 B：docs/关键决策/ 目录 + Agent 提议 + Human 决策

- 在 `docs/` 下新增 `关键决策/` 目录
- 任务总结阶段（Phase 5），Agent 提议哪些 ADR 具有跨任务参考价值
- 使用 `question` 工具询问 Human 是否晋升
- Human 确认后，将 ADR 复制/移动到 `docs/关键决策/`

✅ 符合 Human 的"晋升机制不明确先主动询问"要求
✅ 不需要自动化的晋升算法（避免误判）
✅ 渐进式——先建目录，后续再优化晋升标准

### 方案 C：自动晋升（按引用次数）

- 统计 ADR 被其他 ADR 引用的次数
- 引用超过阈值自动晋升

❌ 实现复杂，V1.8 不现实

## 决策

**选择方案 B**：`docs/关键决策/` 目录 + Phase 5 人工决策晋升。

### 流程

```
1. 任务执行中 → ADR 创建在 tasks/<id>/adrs/（不变）
2. Phase 5 总结时 → Agent 扫描任务中的 ADR，评估跨任务价值
3. Agent 使用 question 工具向 Human 提议晋升特定的 ADR
4. Human 确认 → ADR 复制到 docs/关键决策/ADR-<原标题>.md
5. docs/关键决策/ 中的 ADR 被 Agent 持续引用
6. 未来：废弃/更新的 ADR 在 frontmatter 中标记 status: superseded
```

### 目录结构

```
.self-workflow/docs/
├── 实施经验/
├── 参考模式/
├── 错误经验/
└── 关键决策/               # ← 新增
    └── ADR-xxx-xxx.md      # 跨任务的重大决策记录
```

### 晋升评估标准（Agent 参考）

Agent 在 Phase 5 评估 ADR 是否应晋升时，考虑：
- **跨任务性**：该决策是否影响多个任务/多个模块？
- **框架级**：该决策是否涉及框架的架构、规范、约束？
- **持久性**：该决策是否长期有效（vs 临时解决方案）？
- **可引用性**：其他任务是否可能需要引用此决策？

评估后使用 `question` 工具向 Human 提议，由 Human 决策。

## 理由

1. **解决真问题**：29 个 ADR 埋没在任务目录中，可检索性为零
2. **不过度自动化**：人工决策避免了"自动晋升垃圾"的风险
3. **与经验目录统一**：`关键决策/` 作为 docs/ 的第四个分类，Plugin 自动索引
4. **渐进优化**：先有目录和人工流程，后续 V1.11（Agent 自主决策）可在此基础上优化

## 影响

- `docs/README.md` 需新增 `### 关键决策/` 分类条目
- Phase 5 checklist 增加 "检查是否有可晋升的 ADR" 步骤
- `decision-record-spec`（F4）中描述此晋升机制
