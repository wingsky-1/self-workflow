## 经验提炼

### 经验 1：安装器执行路径必须从项目根目录

| 字段 | 内容 |
|------|------|
| **类型** | 踩坑记录 |
| **等级** | draft |
| **适用工作流** | feat / debug |

**问题背景**

安装器 `index.js` 使用 `process.cwd()` 作为默认目标路径。从 `packages/installer/` 目录执行时，所有文件被写入 `packages/installer/.opencode/` 和 `packages/installer/.self-workflow/`，而非项目根目录下的正确位置。

**解决方案**

始终从项目根目录执行安装器：`node packages/installer/index.js init`

**注意事项**

- 在 `feat-workflow.md` 的快速入门中没有强调安装器的执行路径，后续可补充
- 或者让安装器自动检测项目根目录（通过检测 `task.yaml` 或 `.self-workflow/` 的存在）

### 经验 2：设计阶段的"质疑节点"有价值

| 字段 | 内容 |
|------|------|
| **类型** | 解决方案 |
| **等级** | draft |
| **适用工作流** | feat |

**问题背景**

进入方案设计前先输出质疑报告，这个强制步骤在 V1.5 自身实施中确认了价值——设计文档交付前自己先批判一遍，减少了 Gate 审查中发现的低级问题。

**解决方案**

在阶段 2 入口强制插入质疑节点，包含方向质疑、约束检查、风险提示三项。

**注意事项**

- 质疑报告不需要复杂，3-5 条即可
- 质疑后的"确认"步骤必须执行（不能跳过）
