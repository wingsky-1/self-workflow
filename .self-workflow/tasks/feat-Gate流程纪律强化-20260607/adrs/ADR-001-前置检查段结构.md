# ADR-001：前置检查段的结构与位置

## 背景

V1.21 需要在每个 Gate 入口增加"前置检查"段——将量化输出、产物自查、自检清单、程序化验证声明固化为显式步骤。需决定此段的**位置**（嵌入在审查步骤内 vs. 独立段）和**结构**（列表 vs. 表格 vs. 段落）。

## 备选方案

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A：独立段（前置检查在审查步骤之前） | 前置检查作为 `### Gate重量量化` 之后的独立 `#### 前置检查` 段，不嵌套在任何"步骤"编号下 | 即使 weight=skip 也可见（不依赖审查步骤的存在）；结构清晰，Agent 无法"跳过整节审查步骤"的同时跳过前置检查 | Gate 段增加一个小标题层级 |
| B：步骤 0（嵌套在审查步骤内） | 前置检查作为 `#### 步骤 0：前置检查`，放在现有步骤 1 之前 | 与现有步骤编号体系一致；改动最小 | weight=skip 时容易被连带跳过（"跳过所有审查步骤"包括步骤 0）；与步骤语义混淆（"前置检查"不是"审查"，是审查的入场条件） |
| C：嵌入量化段（不独立） | 在现有 `### Gate重量量化` 段末尾追加自查清单 | 改动量最小，不增加新标题 | 量化段语义不匹配；自查清单与量化公式混杂，可读性差 |

## 选择

**方案 A：独立段**

## 理由

1. **语义正确**：前置检查是"入场条件"而非"审查步骤"——它告诉 Agent "在进入审查之前必须先做这些事"。不应与审查步骤混淆。
2. **防跳过**：weight=skip 时"跳过所有审查步骤"的逻辑不会连带跳过独立段的前置检查。这直接对应 AC-4（即使 weight=skip 也不可省略量化输出）。
3. **可读性**：独立段带明确标题，Agent 阅读时无法"没看到"——需要主动跳过一整节。
4. **与现有结构兼容**：现有 Gate 已有 `### Gate重量量化` 独立段，在其后加 `#### 前置检查` 层级自然。

## 后果

- 每个 Gate 增加一个 `#### 前置检查` 子标题，约 10-15 行内容
- feat-workflow.md 总长度增长约 80-100 行（4 个 Gate × ~20 行 + review-agent 提示词更新）
- 前置检查段内用编号列表（1-4），不用表格（适配 Agent 阅读习惯——列表格式比表格更易逐项检查）

## 关联

- 关联需求：AC-2（产物自查）、AC-3（自检清单）、AC-4（量化输出前置）、AC-6（程序化验证跳过声明）
- 关联 ADR：无
