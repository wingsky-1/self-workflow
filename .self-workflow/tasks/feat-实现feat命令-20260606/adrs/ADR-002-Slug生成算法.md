# ADR-002：Slug 生成算法

| 字段 | 值 |
|------|-----|
| ID | ADR-002 |
| 状态 | 已选择 |
| 决策者 | Agent (Sisyphus) |
| 日期 | 2026-06-06 |

## 背景

workflow-id 格式为 `feat-<slug>-<YYYYMMDD>`，slug 由用户输入的中文描述生成。需要处理 Unicode 字符、特殊符号、长度限制和冲突。

## 备选方案

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A — 纯 ASCII 拼音 | 中文转拼音 | 无 Unicode 兼容问题 | 不可读；依赖外部库；多音字歧义 |
| B — 仅保留 ASCII | 中文直接删除 | 简单、无歧义 | 完全丢失语义 |
| C — 保留 CJK 基本区 | 中文字符保留，其余→`-` | 可读、可排序、中文友好 | Windows 路径需 `-LiteralPath` |
| D — hash-based | SHA256 前 8 位 | 无冲突、纯 ASCII | 完全不可读 |

## 选择

方案 C

## 理由

1. 中文是项目主要语言
2. 产物目录仅在 `.self-workflow/tasks/` 内部使用
3. 已有项目目录使用了相同约定
4. PowerShell 完全支持 Unicode 文件名

## 算法

```
1. 遍历字符：CJK (U+4E00–U+9FFF, U+3400–U+4DBF) → 保留；ASCII a-z,0-9 → 保留（大写转小写）；其余 → '-'
2. 压缩连续 '-'
3. 去除首尾 '-'
4. 截断至 40 字符（优先在 '-' 处截断）
5. 若结果为空 → 使用 "task"
```

## 后果

- 正面：中文可读；可排序
- 负面：外部工具可能不兼容 Unicode 路径。缓解：仅内部目录使用
