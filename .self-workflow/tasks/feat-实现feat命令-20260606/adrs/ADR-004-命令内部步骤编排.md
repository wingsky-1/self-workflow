# ADR-004：命令内部步骤编排

## 背景

`/feat` 命令定义包含 7 个步骤（0-6），需要在 OpenCode 环境中按顺序执行。

## 备选方案

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A — 声明式步骤清单 | Markdown 列出步骤，Agent 逐步骤读取并执行 | 人类可读；与现有命令风格一致 | 依赖 Agent 正确解释自然语言 |
| B — 脚本化执行 | JavaScript/PowerShell 封装所有步骤 | 确定性高；无解释歧义 | 命令文件变为薄包装器；与项目风格不一致 |

## 选择

方案 A

## 理由

1. 与现有命令 (`/adr`, `/catchup`) 一致
2. OpenCode 的 slash command 机制设计为"Agent 读取并执行 Markdown 指令"
3. 步骤间通过文件系统交接状态（各步骤有明确的输入/输出产物）
4. 方案 B 引入的脚本与项目 Markdown/YAML 为主的风格不一致

## 后果

- 正面：人类可读、可维护
- 负面：步骤间状态共享依赖 Agent 记忆。缓解：文件系统作为状态交接媒介

## 关联

- 关联命令：`.opencode/commands/adr.md`（相同模式）
- 关联需求：F4（进入阶段 1）
