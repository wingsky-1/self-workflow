# ADR-004：安装器管理文件的修改路径规则

> 日期：2026-06-06
> 状态：已采纳（经修正）
> 关联：V1 ADR-004 目录职责划分、AGENTS.md

## 背景

本次任务中，Agent 直接修改了 `.self-workflow/configs/guides/feat-workflow.md`，后续运行安装器 `--force` 时被模板源覆盖，导致修改丢失。

根因：Agent 在自举项目中混淆了"开发层"和"运行时层"——`.self-workflow/configs/` 是安装器管理的编译产物，不是可自由编辑的运行时数据。

## 已经存在的答案

V1 ADR-004 早已定义了目录职责（本应提前阅读）：

| 目录 | 修改方式 |
|------|---------|
| `.opencode/` 全部 | 仅安装器 |
| `.self-workflow/configs/` | 仅安装器 |
| `.self-workflow/tasks/`、`.self-workflow/docs/` | 直接编辑 |

安装器模板源 `packages/installer/templates/` 是这些文件的唯一来源。

## 决策

**约束位置：AGENTS.md，不上 agent-reasoning Skill。**

理由：
1. `agent-reasoning` Skill 是通用 Skill（委托优先、上下文管理、验证结果），承载 Agent 行为模式，不应包含项目特定约束
2. 项目特定约束（安装器管理文件列表、修改路径规则）应放在 `AGENTS.md`
3. V1 ADR-004 已定义框架，本次是对框架的一次**违反**，而非框架本身的缺陷

## 修正记录

初版决策建议在 `agent-reasoning` Skill 的 MUST 规则中加入此约束。经指出后修正：
- Skill MUST #5 已回退
- 约束改为更新 `AGENTS.md` + 完善 `.self-workflow/docs/installer-错误经验.md`

## 影响

- `AGENTS.md` 新增第 3 条：`.self-workflow/configs/` 和 `.self-workflow/specs/` 通过安装器模板源修改
- `.self-workflow/docs/installer-错误经验.md` 重写为完整踩坑记录（含自举项目特殊性分析）
- 不修改 `agent-reasoning` Skill
