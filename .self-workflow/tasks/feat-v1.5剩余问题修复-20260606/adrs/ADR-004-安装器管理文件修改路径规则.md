# ADR-004：安装器管理文件的修改路径规则

> 日期：2026-06-06
> 状态：已采纳
> 关联任务：feat-v1.5剩余问题修复-20260606

## 背景

本次任务中，Agent 直接修改了 `.self-workflow/configs/guides/feat-workflow.md`，后续运行安装器 `--force` 时被模板源覆盖，导致修改丢失。

根因不是安装器设计问题，而是 **Agent 不知道哪些文件由安装器管理、必须通过模板源修改**。

## 已安装器管理的文件

（定义在 `packages/installer/index.js` 的 MANIFEST 中）

| 运行时路径 | 模板源路径 |
|-----------|-----------|
| `.opencode/*` | `packages/installer/templates/skills/`、`agents/`、`commands/` |
| `.self-workflow/configs/*` | `packages/installer/templates/configs/` |
| `.self-workflow/specs/*` | `packages/installer/templates/specs/` |

**直接修改的文件**（不在 MANIFEST 中）：
- `.self-workflow/tasks/` — 任务产物
- `.self-workflow/docs/` — 经验文档

## 决策

在 `agent-reasoning` Skill 的 MUST 规则中新增第 5 条：

> **修改文件前检查安装器**：修改 `.opencode/`、`.self-workflow/configs/`、`.self-workflow/specs/` 下的任何文件前，必须先判断：该文件是否有对应的安装器模板源（`packages/installer/templates/`）。如有，修改模板源 → 运行安装器同步；如无，直接修改。

## 理由

1. Agent 行为约束比安装器设计改动更根本——即使安装器改了，Agent 不知道规则还是会踩坑
2. `agent-reasoning` Skill 是每个 Agent 任务启动时加载的，覆盖面最广
3. 规则放在 MUST 段确保 Agent 不会忽略

## 影响

- `packages/installer/templates/skills/agent-reasoning/SKILL.md` 新增 MUST 第 5 条
- 所有后续 Agent 在修改文件前会检查是否属于安装器管理范围
- 已知的 `.self-workflow/docs/installer-错误经验.md` 记录了这个陷阱的细节
