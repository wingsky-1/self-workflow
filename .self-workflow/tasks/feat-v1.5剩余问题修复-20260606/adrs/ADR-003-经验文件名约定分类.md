# ADR-003：经验分类 — 文件名约定

> 日期：2026-06-06
> 状态：已采纳
> 关联：`.self-workflow/docs/经验分级与加载指引.md`

## 背景

`.self-workflow/docs/` 当前仅有 `*-实施经验.md` 一种模式，所有经验混在一个分类中。`docs/todo.md` #9 要求增加分级（实施经验/参考模式/错误经验），并设计渐进式披露机制让 Agent 在正确时机加载正确的经验文档。

## 备选方案

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A：文件名约定 | `<领域>-实施经验.md`、`<领域>-参考模式.md`、`<领域>-错误经验.md` | Agent 扫描文件名即可判断分类；无需解析文件内容 | 跨分类经验需多文件或复合文件名 |
| B：frontmatter 字段 | 文件内 YAML frontmatter 定义 `category: 实施经验` | 一个文件可包含多个分类（按节） | 需解析文件内容才能获取分类；Agent 扫描成本高 |
| C：目录分类 | `docs/实施经验/`、`docs/参考模式/`、`docs/错误经验/` 子目录 | 文件系统级分类 | 交叉分类时难以放置；目录层级增加 |

## 决策

**选择方案 A（文件名约定）**。

文件命名：`<领域>-<分类>.md`（如 `feat-command-实施经验.md`、`installer-参考模式.md`、`路径处理-错误经验.md`）。

## 理由

1. **Agent 扫描成本最低**：文件名匹配是字符串比对，无需读取文件内容——在上下文窗口受限的场景中至关重要
2. **渐进式披露的基础**：只有文件名可被低成本扫描，分类信息才能在实际检索中被利用
3. 交叉分类场景处理：优先拆分为多文件；高度相关时可使用复合文件名（如 `*-实施经验+错误经验.md`）

## 影响

- 新建设计文档 `.self-workflow/docs/经验分级与加载指引.md`
- `feat-workflow.md` Phase 5 新增 doc 级经验判断标准和写入指引
- 渐进式披露机制：任务启动扫描文件名 → 设计阶段检索参考模式 → 遇到错误检索错误经验 → Phase 5 写入对应分类
