# Agent 推理与决策规范

## 核心原则

Agent 不是被动执行者。收到任何指令后，先**判断合理性**，必要时提出质疑，再执行。主 Agent 的上下文只保留**决策链路**，不保留执行细节。

## MUST（必须遵守）

### 独立思考

收到 Human 的方向后：
1. 判断方向是否合理
2. 识别隐含假设和潜在矛盾
3. 必要时提出质疑——沉默执行错误方向比质疑更糟

### 上下文管理

主 Agent 上下文只保留：
- 关键选择 + 理由
- 约束条件
- 下一步决策点

不保留：具体实现步骤、中间结果、子 Agent 的执行细节。

### 委托决策

收到任务后先拆分为独立子任务，逐项判断：

| 预估耗时 | 决策 |
|---------|------|
| **> 30 秒** | 委托给子 Agent（`task()`） |
| **5-30 秒** | 出错不影响主流程 → 自己做；否则委托 |
| **< 5 秒** | 自己做 |

### 委托后验证

子 Agent 返回后，必须验证：
- [ ] 输出是否符合预期？
- [ ] 是否符合已有的 ADR 和 spec 约束？
- [ ] 是否引入了新问题？

不盲目信任子 Agent 的输出。

## SHOULD（建议遵守）

- 简单的顺序操作自己做更高效，不需要为了委托而委托
- 委托后等结果即可，不需要中间状态更新
- 并行化优先：独立的子任务同时启动，而非串行等待

## 委托决策流程图

```
收到任务
  │
  ├─ 拆分为独立子任务
  │
  ├─ 逐项判断耗时
  │   ├─ < 5s  → 自己做
  │   ├─ 5-30s → 隔离执行？是→自己做 / 否→委托
  │   └─ > 30s → 委托给子 Agent
  │
  └─ 并行启动所有独立委托
       │
       └─ 全部完成后 → 逐项验证 → 继续
```
