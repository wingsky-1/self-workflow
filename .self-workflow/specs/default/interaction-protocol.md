# 交互协议规范

## 核心原则

Agent 在与用户交互时，优先使用结构化问答工具（`question`）替代纯文本列举选项。

## MUST（必须遵守）

- **选项选择**：涉及 2 个以上选项供用户选择时，必须使用 `question` 工具，而非纯文本列举。
- **确认/继续/拒绝**：所有"是否确认？""是否继续？""是否同意？""是否拒绝？"类提问必须使用 `question` 工具——即使看起来是"简单确认"的对话场景也不例外。
- **偏好选择**：让用户在多个方案中做选择时使用 `question` 工具。
- **方向确认**：Human 给出方向性指令后（如"方向认可，继续"），Agent 若需确认具体方案或下一步，仍需使用 `question`。
- **总结先行**：询问用户是否继续下一步前，先总结已完成的内容，再说明下一步要做什么。
  - 格式：`已完成：X、Y、Z → 下一步：做 A（涉及 B、C、D）`
- **选项描述具体**：`question` 的 `description` 字段说明选择带来的后果，而非笼统描述。

## SHOULD（建议遵守）

- **开放式提问**（"还有什么？"、"有什么想法？"）不需要使用 `question` 工具。
- **需要自由输入的场景**不使用 `question`。

## 示例

```
✅ 正确：用 question 工具，options 含具体描述
❌ 错误：纯文本列举"您想选择方案 A 还是方案 B？"
```
